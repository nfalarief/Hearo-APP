# Hearo AI — Documentation

This directory contains supporting documentation for **Hearo AI**, an AI-driven mobile application designed to improve communication accessibility for Deaf and hard-of-hearing individuals.

## 📚 Available Documentation

The documentation in this directory may include:

* **Research Paper** — Academic research related to the development of Hearo AI using a Design Thinking approach.
* **UI/UX Case Study** — Documentation of the complete design process, from problem identification to final interface design.
* **Presentation** — Project presentation materials used for academic, research, or competition purposes.
* **Usability Testing** — Evaluation materials and findings from user testing.
* **Design Process** — Supporting materials such as user flows, wireframes, information architecture, and design iterations.

## 🔬 Research Background

Hearo AI is associated with the research:

**“A Design Thinking Approach to Developing an AI-Driven Mobile Experience for Deaf Inclusivity”**

The research explores how artificial intelligence and human-centered design can contribute to more inclusive digital experiences for Deaf and hard-of-hearing individuals.

## 🧠 Design Methodology

The project follows the **Design Thinking** framework:

1. **Empathize** — Understanding user needs, communication barriers, and accessibility challenges.
2. **Define** — Identifying and defining the primary problems experienced by users.
3. **Ideate** — Generating potential AI-driven and accessibility-focused solutions.
4. **Prototype** — Developing wireframes and high-fidelity interactive designs.
5. **Test** — Evaluating usability and refining the design based on feedback.
